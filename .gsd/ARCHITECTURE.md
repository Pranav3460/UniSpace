## Overview

CampusConnect is a full-stack mobile application built with React Native (Expo) and a Node.js Express backend. It facilitates campus communication, resource sharing, study groups, notices, and event management. The system relies on MongoDB for data persistence, Cloudinary for media/file storage, and Socket.IO for real-time updates.

## Data Flow

1. **Client Request:** The mobile app makes REST API calls using Axios from `src/api/client.ts`.
2. **Backend Processing:** Express handles the routes (`/api/...`), processing the payload, authenticating, and authorizing users (with `bcrypt`).
3. **Database Operations:** The server queries or mutates data using Mongoose models (Notice, LostFound, Resource, Event, Group, User).
4. **External Media:** If the request includes file uploads, the backend (or frontend directly via `src/lib/upload.ts`) delegates to Cloudinary and stores the resulting secure URL in MongoDB.
5. **Real-Time Notification:** Successful mutations often trigger an `io.emit` broadcast over Socket.IO to connected clients to instantly refresh UI state.
